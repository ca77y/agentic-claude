# Harden the researcher's evidence discipline

- [x] Harden the researcher's evidence discipline #improvement ⏫ 🆔 harden-researcher-evidence-discipline #important
  - The `researcher` has no rule for distinguishing a broken tool from a genuine absence of evidence, and no rule for reading a source repository across time. Both produce confidently wrong research, repeatedly.
  - Background — search faults read as evidence of absence: in a five-way parallel research fan-out, the web search tool returned an empty array for every query from one agent, across `google`, `brave`, and `duckduckgo` engine overrides. An empty array is not an error, so agents read it as "no such thing exists". One agent's conclusion ("no migration API") rested partly on a zero-hit search. In a later four-way fan-out the same fault hit three of four agents; two of three ran a control query (a term that cannot legitimately return zero, e.g. `typescript`), got an empty array back, and correctly downgraded every absence-based conclusion to "unretrieved, not absent" — the third did not, and reported search-blocked gaps as findings.
  - Background — fallbacks get rediscovered per agent: in that same fan-out, two agents independently converged on the same two workarounds after roughly eight wasted calls each. (1) The Hacker News Algolia API fetched directly as JSON — `https://hn.algolia.com/api/v1/search?query=<q>&tags=story` to find threads, then `https://hn.algolia.com/api/v1/items/<objectID>` for whole comment trees. (2) `https://lite.duckduckgo.com/lite/?q=<query>` fetched as a page, which works as a general search proxy where the regular `duckduckgo.com/html` endpoint returns an anti-bot page. Also established: Reddit's `.json` API is 403-blocked, while `old.reddit.com` thread pages fetch fine but are very token-expensive. A third agent found neither and reported gaps the other two had already routed around.
  - Background — repositories read only at HEAD: a pass checked a vendor's infrastructure template at HEAD, found no `wal_level` override, and dismissed dated user reports of the setting reverting after restart as "should work in theory, unverified in practice" — leaving a real contradiction unresolved across two passes. The reports were true: one commit had introduced a hardcoded `wal_level = replica`, and a later vendor commit removed it with the message "Forcing wal_level=replica clobbered logical-replication users on restart". Reading HEAD alone cannot see a defect that existed in the window the reports came from, and version-pinned container images mean users keep running the pre-fix build long after the fix lands. Two of three parallel agents inspected HEAD only and reached the wrong conclusion; the one that searched commit history found the answer immediately.
  - Scope: `plugins/ca77y-engineering/agents/researcher.md`. Keep the rules tool-agnostic — name the failure signature and the response, not a specific MCP server's schema.
  - Acceptance criteria:
  - An empty web-search result is defined as a suspected tool fault, never as evidence of absence.
  - On an empty result the researcher issues one control query whose term cannot legitimately return zero, and uses the outcome to label every absence-based conclusion in its report as either "confirmed absent" or "unretrieved, not absent".
  - The known-good fallbacks are named in the definition so they are read rather than rediscovered: the HN Algolia search and item endpoints, the DuckDuckGo `lite` endpoint fetched as a page, and `old.reddit.com` thread pages with the `.json` API noted as blocked.
  - The researcher reports which fallback it used whenever the primary search failed.
  - When a vendor's current source contradicts dated practitioner failure reports, the researcher treats the disagreement as a timeline question rather than a credibility question: it searches commit history and blame for the parameter over the window the reports span, and checks whether the deployed artifact auto-upgrades or is version-pinned, before concluding user error.
  - See [`../ARCHITECTURE.md`](../ARCHITECTURE.md) for the self-improvement channel these observations arrived through.
